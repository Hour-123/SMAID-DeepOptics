## 3. 与其他运行结果的对比

### 3.1 对比 `directpsf_16layers_p191_20260307_233408`

参考结果：

- `DirectPSF` 最优验证总损失：`0.11055`
- 本次 `PhaseMask` 最优验证总损失：`0.10849`

对比结论：

- 本次 `PhaseMask` 的最优验证损失略优于 `DirectPSF 16-layer / 191`，改善约 `1.9%`。
- 本次训练速度也更快：
  - `PhaseMask` 训练吞吐约 `3.20 it/s`
  - `DirectPSF 16 x 191` 训练吞吐约 `2.21 it/s`
  - `PhaseMask` 验证吞吐约 `7.58 it/s`
  - `DirectPSF 16 x 191` 验证吞吐约 `5.64 it/s`

这个结果的意义不是说物理相位板传播一定“更便宜”，而是当前对比对象用了 `16` 个深度层，而本次 `PhaseMask` 用的是 `8` 个深度层，因此在现有配置下，本次方案同时拿到了更低的验证损失和更高的吞吐。

### 3.2 对比更早的 `phase_mask_v151_formal_20260309_120210`

参考结果：

- `phase_mask_v151_formal_20260309_120210` 最优验证总损失：`0.46391`
- 本次 `phase_mask_formal_20260309_123854` 最优验证总损失：`0.10849`

对比结论：

- 验证总损失下降约 `76.6%`。
- 这表明当前版本的 `phase_mask_formal` 已经不是一个“能跑通但效果弱”的实验，而是一个真正具备竞争力的相位板训练结果。

需要注意：

- 这两个 phase-mask 运行并非完全同配。
- 早期 `v151_formal` 只训练了 `5` 个 epoch，且 `psf_size=127`、`psf_edge_reg=0`。
- 当前正式结果使用了 `40` 个 epoch、`psf_size=191`、并引入了 `psf_edge_reg=1e-3`。

因此，更合理的解读是：当前工程版本、训练时长和正则化组合已经显著改善了 phase-mask 路线的可训练性。

## 4. 损失曲线解读

从 `logs/history.json` 可以看到：

- 训练损失从 `1.0166` 持续下降到 `0.0866`。
- 验证损失从 `1.0463` 持续下降到 `0.1085`。
- 中间有几次明显波动，尤其在 `epoch 18`、`epoch 28`、`epoch 36` 附近验证损失短时回升。
- 这些回升之后模型都能重新回到更低区间，说明不是发散，而更像是联合优化中的阶段性重构。

这说明：

- 相位板和重建网络的联合训练是可控的。
- 当前学习率和损失权重没有导致训练崩溃。
- 训练到第 `39` 轮时仍在变好，因此这套配置还没有被充分榨干。

## 5. PSF 统计变化

从 `history.json` 中的 PSF 统计看，模型从一个较尖锐的初始核逐步学习到更分散、更高熵的深度相关 PSF 族：

- `psf_peak_mean`：`0.1174 -> 0.1027`
- `psf_center_3x3_mass_mean`：`0.4102 -> 0.2945`
- `psf_entropy_mean`：`4.1308 -> 5.9919`
- `psf_second_moment_mean`：`154.4 -> 1124.8`
- `psf_edge_energy_mean`：`0.00619 -> 0.05367`
- `psf_outside_center21_max_mean`：最终约 `0.00115`

解释如下：

1. `peak_mean` 和 `center_3x3_mass_mean` 明显下降。
   这说明 PSF 不再停留在近似 delta 的状态，而是把更多能量从中心释放到更大范围。

2. `entropy_mean` 明显升高。
   这说明 PSF 形状变得更复杂、更有编码性，而不是只做一个尖峰模糊。

3. `second_moment_mean` 比初始值大很多。
   说明最终核的空间扩展半径显著增加，系统在主动利用“可辨别的模糊结构”承载深度信息。

4. `outside_center21_max_mean` 始终很小。
   这意味着虽然 PSF 变复杂了，但没有明显出现“孤立远端尖峰主导”的异常结构；能量分布仍然相对连续。

5. `edge_energy_mean` 在中期一度较高，最终收敛到 `0.0537`。
   这说明边缘正则确实起作用了，但它并没有把 PSF 压回极端中心化，而是在允许编码能力和约束边缘泄露之间做折中。

## 6. 训练过程中的一个关键现象

`epoch 28` 是一个值得特别记录的拐点：

- `psf_peak_mean` 暂时降到 `0.0940`
- `psf_center_3x3_mass_mean` 暂时降到 `0.2671`
- `psf_entropy_mean` 暂时升到 `6.5356`
- 同期验证损失升到 `0.1917`

这通常意味着模型一度把 PSF 推向了一个“编码很强、但解码难度也更高”的区域。随后：

- `epoch 29` 验证损失回落到 `0.1299`
- 最终 `epoch 39` 到达 `0.1085`

因此可以推断：

- 当前优化过程确实在探索不同的光学编码强度。
- 最终最优点并不是“最发散、最扩散”的 PSF，而是一个更平衡的状态。

## 7. 高度图统计解读

来自 `height_stats_history.json`：

- `height_mean`：`7.79e-7 -> 7.90e-7`
- `height_std`：`9.23e-8 -> 2.03e-7`
- `height_min`：从 `3.20e-7` 逐步接近 `0`
- `height_max`：很快逼近 `1.55e-6`
- `height_dynamic_range`：最终约 `1.55e-6`

这说明：

1. 高度图在训练后期几乎用满了允许的动态范围。
   也就是最小值接近 `0`，最大值接近 `height_map_max`。

2. 高度图标准差明显上升。
   说明相位板不再是接近平坦面的弱扰动，而是学习到了更强的形貌变化。

3. 当前约束更像“盒约束”而不是“软引导”。
   因为模型最终把可用高度窗口基本打满了，这通常意味着现有任务会积极利用更大的相位调制能力。

这是一把双刃剑：

- 正面：说明 phase-mask 路线确实在学习有效光学编码，而不是网络单独承担全部任务。
- 风险：如果真实制造工艺对边缘陡变、厚度极值或装调偏差敏感，那么这种接近饱和的高度分布可能在实物中更脆弱。

## 8. 量化正则的作用

量化正则从 `1e-4` 线性升到 `1e-3`，最终光学正则损失约 `5.37e-5`。

这说明：

- 当前量化约束是“温和牵引”，不是主导项。
- 它足以把连续高度图往量化网格上轻推，但没有明显破坏主任务收敛。
- 这对后续导出 `continuous` / `quantized` 两套结果是有意义的。

但这里也有一个局限：

- 当前目录中没有保留下来的 `phase/`、`psf/continuous/`、`psf/quantized/`、`im/continuous/`、`im/quantized/` 导出文件。
- 所以这次报告只能确认“代码和日志层面已经支持量化比较”，还不能进一步检查连续版与量化版在视觉质量上的真实差距。

## 9. 当前结果说明了什么

这次运行已经说明：

- `PhaseMaskOptics` 在当前代码里不是占位实现，而是能稳定训练并达到接近当前最佳水平的主路线之一。
- 物理相位板参数化没有妨碍优化，反而提供了比大自由度 `DirectPSF` 更规整、也更有物理意义的编码方式。
- 现有正则化组合是有效的，既避免了明显的异常尖峰，又保留了足够的深度编码能力。

同时，这次结果也说明：

- 当前评估主要还是训练/验证损失，还没有单独输出 PSNR、SSIM、RMSE、AbsRel 等更标准的任务指标。
- 由于缺少可视化导出文件，无法在这份报告中直接检查：
  - 连续高度图与量化高度图的空间形貌差异
  - 连续 PSF 与量化 PSF 的视觉差异
  - 最佳样本的 RGB 重建质量与深度边界质量

## 10. 结论

如果只根据当前仓库中可见的日志与 summary 来判断，这次 `phase_mask_formal_20260309_123854` 可以视为目前项目里最有说服力的 phase-mask 正式结果之一：

- 它收敛稳定。
- 它在验证损失上已经略优于 `DirectPSF 16-layer / 191`。
- 它比更早的 phase-mask 正式结果强得多。
- 它学到的不是退化成中心尖峰的伪相位板，而是一组明显更高熵、更有深度编码能力的 PSF。

当前最合理的总体判断是：

- `PhaseMask` 路线已经从“可行性尝试”进入“值得作为主线继续推进”的阶段。

## 11. 建议的下一步

1. 保留当前超参组合作为正式基线。
   这组参数已经证明 phase-mask 能收敛到有效解。

2. 补全标准评估指标。
   建议在验证阶段增加 `PSNR`、`SSIM`、`RMSE`、`AbsRel`，避免只看训练损失。

3. 恢复或重新导出 `im/`、`psf/`、`phase/` 产物。
   当前目录只有日志，没有最终图像和相位图，不利于进一步判断连续版与量化版的真实差距。

4. 继续关注制造鲁棒性。
   当前高度图几乎用满动态范围，下一步值得测试：
   - 更强的平滑约束
   - 更强的量化约束
   - `height_map_noise_std > 0` 的制造误差鲁棒训练

5. 在 phase-mask 主线下再做少量系统性 ablation。
   优先推荐：
   - `num_depth_layers = 8` vs `16`
   - `psf_size = 127` vs `191`
   - `psf_edge_reg = 0` vs `1e-3`
   - 开关 `quantization_reg_anneal`
