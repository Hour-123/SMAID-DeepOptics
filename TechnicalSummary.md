# SMAID: Single-shot Monocular All-in-focus Imaging and Depth-sensing System - 技术报告

> **项目名称**: SMAID (单次曝光单目全聚焦成像与深度感知系统)
> **研究方向**: 计算摄像学、深层光学、单目 RGB-D 成像
> **目标定位**: 在一次曝光、一个镜头、一个二维传感器的约束下，同时恢复全聚焦 RGB 图像与深度图
> **文档用途**: 作为 `SMAID-DeepOptics` 项目的总体技术说明、实现蓝图与后续代码开发依据

---

## 1. 项目目标与核心问题

### 1.1 问题背景

传统成像系统在以下三个目标之间存在天然冲突：

1. **大光圈与长景深冲突**
   - 大光圈带来更高进光量和更好信噪比，但景深变浅，焦外区域模糊严重。
   - 小光圈能够扩展景深，但会牺牲通光量，使暗光场景噪声更强。

2. **单次曝光与三维感知冲突**
   - 普通二维传感器只能记录空间辐照度积分结果，无法直接获取场景深度。
   - 单目深度估计如果只依赖语义先验，容易出现尺度漂移与泛化不稳定问题。

3. **高质量成像与轻量硬件冲突**
   - 主动深度方案如 ToF、结构光增加功耗与系统复杂度。
   - 双目/多目方案增加体积、标定难度和遮挡问题。
   - 光场系统存在空间分辨率与角分辨率之间的显著折中。

### 1.2 SMAID 的核心思想

SMAID 采用**光学-算法联合设计**思路，在镜头孔径处引入一个可优化的**相位掩模**，使系统的点扩散函数 `PSF_z` 随深度变化，并将这种变化编码到单次曝光图像中。随后用神经网络从该编码图像中联合恢复：

- **All-in-focus RGB**
- **Metric / Relative Depth**

换句话说，SMAID 不是把模糊视为噪声，而是把模糊设计成**带有可解码深度信息的物理编码**。

### 1.3 期望解决的问题

SMAID 希望在一个统一系统中同时解决以下问题：

- 从单张带有深度相关模糊的图像中恢复接近全聚焦的清晰图像。
- 利用不同景深层对应的模糊结构估计场景深度。
- 在保持较大光圈和较高进光量的前提下，突破传统光学系统中景深与信噪比的折中。
- 为后续真实相位板制造和实拍验证预留可落地的物理约束。

## 2. 与 `DeepOpticsHDR` 的关系

### 2.1 为什么要参考 `DeepOpticsHDR`

`DeepOpticsHDR` 已经验证了一条成熟的技术路径：

- 使用可学习相位板作为物理编码器；
- 使用 Fresnel 衍射建立可微光学前向模型；
- 使用卷积网络作为电子解码器；
- 对光学参数与网络参数进行端到端联合优化。

这条路线与 SMAID 的研究范式高度一致，因此 `DeepOpticsHDR` 可以视作 SMAID 的**代码参考底座**。

### 2.2 共同点

两个项目共享以下基础思想：

1. **统一的编码器-瓶颈-解码器框架**
   - 编码器：相位掩模及其产生的 PSF
   - 瓶颈：单次曝光二维传感器
   - 解码器：深度网络

2. **统一的物理建模基础**
   - 相位掩模高度图参数化
   - Fresnel / 角谱传播
   - 频域卷积与 OTF 表示
   - 光学可制造性约束

3. **统一的优化方式**
   - 光学参数与网络参数联合训练
   - 使用可微分的前向成像过程
   - 通过正则项约束相位板形貌

### 2.3 核心差异

| 维度 | `DeepOpticsHDR` | `SMAID` |
| --- | --- | --- |
| **目标任务** | 单次曝光 HDR 重建 | 单次曝光全聚焦重建 + 深度估计 |
| **PSF 特性** | 基本上是场景无关的单一 PSF | 明显依赖深度的 `PSF_z` |
| **前向模型** | 单一 PSF 卷积 + 饱和裁剪 | 多深度层分解 + 深度相关卷积求和 |
| **网络输出** | HDR 图像 | RGB 图像 + 深度图 |
| **损失函数** | HDR 重建为主 | 多任务联合损失 |
| **训练数据** | HDR 图像 | RGB-D 图像对 |

### 2.4 对 SMAID 的直接启发

从实现角度看，SMAID 并不是从零起步，而是在 `DeepOpticsHDR` 基础上向以下方向扩展：

- 从**单一 PSF**扩展为**深度相关 PSF 集合**；
- 从**单头图像重建网络**扩展为**双头 RGB-D 网络**；
- 从**单任务损失**扩展为**多任务联合损失**；
- 从**二维强度恢复问题**扩展为**几何感知问题**。

## 3. 系统整体架构

### 3.1 总体流程

```text
全聚焦 RGB + 深度图
        ↓
深度离散化 / 分层表示
        ↓
深度相关光学编码器 (PSF_z)
        ↓
传感器积分 + 噪声 + 裁剪
        ↓
单张编码图像
        ↓
共享编码器网络
        ↓
RGB 解码头 + Depth 解码头
        ↓
输出全聚焦 RGB 与深度图
```

### 3.2 三个核心组成

#### 3.2.1 物理编码器

SMAID 的物理编码器由相位掩模和成像传播模型构成。其目标不是简单减弱模糊，而是让不同深度平面的点目标在传感器上形成**彼此可区分的模糊模式**。

#### 3.2.2 物理瓶颈

瓶颈是单次曝光下的二维传感器测量。由于传感器只记录强度分布而不记录波前相位，来自不同深度的信息在此被压缩到一张图像中。

#### 3.2.3 电子解码器

电子解码器需要完成两个任务：

- 从编码模糊中恢复结构清晰的全聚焦图像；
- 从模糊模式差异中恢复像素级深度。

因此它必须同时具备：

- 去模糊能力；
- 多尺度结构建模能力；
- 深度边界保持能力；
- RGB 与深度任务之间的信息交互能力。

## 4. 光学建模

### 4.1 光学参数化目标

SMAID 中的光学模块需要回答两个问题：

1. **相位掩模如何建模？**
2. **给定某个深度，系统 PSF 如何计算？**

这两个问题共同决定了端到端训练是否可微、是否稳定，以及最终相位板是否可制造。

### 4.2 相位掩模表示

相位掩模通常通过**高度图** `h(x, y)` 参数化。不同位置的厚度差会引入不同相移：

$$
\phi(x,y,\lambda) = \frac{2\pi}{\lambda}(n(\lambda)-1) h(x,y)
$$

其中：

- `h(x, y)` 为相位板高度图；
- `n(\lambda)` 为材料折射率；
- `\lambda` 为波长。

这种表示方式的优点：

- 与实际制造过程对应自然；
- 容易加入高度限制与平滑正则；
- 可直接继承 `DeepOpticsHDR` 的实现方式。

### 4.3 波前传播模型

SMAID 延续可微波动光学建模路线。一个典型计算流程如下：

```python
input_field = ones(...)
field = PhasePlate(wave_lengths, height_map, refractive_indices)(input_field)
field = circular_aperture(field, r_cutoff)
field = apply_defocus_phase(field, depth=z)
field = propagate_fresnel(field, distance=sensor_distance, ...)
psf_z = get_intensities(field)
```

其含义是：

1. 输入波前进入孔径；
2. 相位掩模施加空间变化相位延迟；
3. 考虑某个深度对应的离焦项；
4. 传播到传感器平面；
5. 对复振幅取强度得到 `PSF_z`。

### 4.4 深度相关 PSF 的关键机制

SMAID 的核心不是单一 PSF，而是一个随深度变化的 PSF 家族：

$$
PSF(x, y, z) = \left| \mathcal{F} \left\{ P(u,v)\exp(j\phi(u,v))\exp\left(j\frac{\pi}{\lambda z}(u^2+v^2)\right) \right\}\right|^2
$$

其中：

- `P(u,v)` 是孔径函数；
- `\phi(u,v)` 是相位掩模引入的相位；
- 额外二次相位项反映了与焦平面偏离相关的离焦效应。

设计目标并非让所有深度都成像清晰，而是让：

- 邻近深度的 PSF 有足够差异，以便网络判别；
- 模糊模式在 RGB 解码上仍然可逆，不至于丢失过多频率信息；
- PSF 形状满足真实制造与装调要求。

### 4.5 深度离散化

为了让前向模型可训练且计算可控，连续深度通常被离散为 `K` 个深度层：

```python
depth_planes = discretize_depth(depth_map, num_layers=K)
```

离散化的意义：

- 降低训练时为每个像素单独计算 PSF 的代价；
- 使前向模型可以写成一组分层卷积求和；
- 方便缓存或预计算不同深度层的 `PSF_z`。

常见取值可以从 `K = 8, 16, 32` 试验，根据精度与显存折中确定。

### 4.6 分层卷积前向模型

在离散深度层设置下，传感器图像可表示为：

$$
I_{sensor} = \sum_{k=1}^{K} (I_{rgb} \odot M_k) * PSF_k + \eta
$$

其中：

- `I_rgb` 为全聚焦场景；
- `M_k` 为第 `k` 个深度层的二值或软掩模；
- `PSF_k` 为该深度层对应的 PSF；
- `*` 表示卷积；
- `\eta` 为噪声项。

伪代码如下：

```python
def simulate_smaid_camera(all_focus_rgb, depth_map, phase_mask, num_layers=16):
    depth_planes = discretize_depth(depth_map, num_layers)
    sensor_img = 0.0

    for layer_idx in range(num_layers):
        psf_z = compute_depth_dependent_psf(phase_mask, layer_idx)
        layer_rgb = all_focus_rgb * depth_planes[layer_idx]
        sensor_img += convolve(layer_rgb, psf_z)

    sensor_img = add_sensor_noise(sensor_img)
    sensor_img = clip(sensor_img, 0.0, 1.0)
    return sensor_img
```

这里的计算逻辑与 `DeepOpticsHDR` 中已经存在的 `depth_dep_convolution()` 在思想上高度一致，因此这是最重要的可复用代码入口之一。

### 4.7 传感器退化建模

除模糊外，还需要在前向模型中加入基本的传感器退化：

- 读出噪声；
- 光子噪声；
- 裁剪饱和；
- 可选相机响应曲线；
- 可选量化误差。

这些退化项的作用不是增加复杂度，而是提高网络对真实成像系统的鲁棒性。

### 4.8 光学可制造性约束

如果没有物理约束，优化可能产生不可制造或极端敏感的高度图。因此需要加入以下约束：

1. **高度范围限制**
   - 限制最大厚度，避免相位板超出制造工艺窗口。

2. **平滑性约束**
   - 对高度图使用 Laplacian 正则，抑制局部剧烈振荡。

3. **制造误差鲁棒性**
   - 在训练时对高度图加入微小随机扰动，模拟实际加工误差。

4. **色散一致性**
   - 对 RGB 波长同时建模，避免仅对单一波长有效。

## 5. 成像模型与物理信息流

### 5.1 从场景到传感器的映射

SMAID 的前向成像过程可概括为：

1. 场景提供全聚焦 RGB 与真实深度；
2. 深度被离散为多个层；
3. 每一层通过对应的 `PSF_z` 编码到传感器；
4. 所有层的贡献在传感器上叠加；
5. 叠加结果经过噪声与裁剪，形成网络输入。

### 5.2 物理瓶颈本质

单张编码图像本质上把三维场景压缩为二维观测，因此问题天然是病态的。SMAID 的关键不是完全消除病态，而是通过设计相位掩模，使这种病态在数据分布上变得**可学习、可解码**。

### 5.3 光学设计原则

理想的 `PSF_z` 需要同时满足以下性质：

1. **深度可辨识性**
   - 相邻深度层的 PSF 应具有统计上可分的形状差异。

2. **去模糊可逆性**
   - 不能只编码深度而完全破坏图像频率信息。

3. **边缘友好性**
   - 在物体边界区域，模糊结构应尽量保留空间定位信息。

4. **鲁棒性**
   - 对噪声、制造偏差、轻微装调误差不应过于敏感。

## 6. 网络架构设计

### 6.1 总体设计原则

SMAID 网络推荐采用**共享编码器 + 双解码器头**的多任务结构：

- 共享编码器提取与编码模糊相关的多尺度特征；
- RGB 解码头负责去模糊与细节恢复；
- Depth 解码头负责几何推断；
- 两个任务在中高层特征空间共享信息，在末端分别输出。

### 6.2 输入与输出定义

#### 输入

- 单张编码图像 `x`
- 可选额外输入：
  - 简单反卷积结果；
  - 归一化坐标通道；
  - 预估景深先验。

#### 输出

- `\hat{I}`: 预测的全聚焦 RGB 图像
- `\hat{D}`: 预测的深度图

### 6.3 共享编码器

共享编码器可以沿用 `DeepOpticsHDR` 中已实现的 U-Net 风格编码器，例如中等规模版本：

```text
Input
  ↓
Conv(64) → Conv(64) → Pool
  ↓
Conv(64) → Conv(64) → Pool
  ↓
Conv(64) → Conv(64) → Pool
  ↓
Conv(64) → Conv(64) → Pool
  ↓
Bottleneck
```

编码器需要重点提取：

- 模糊尺度信息；
- 模糊方向性与纹理信息；
- 边缘结构；
- 可能与深度相关的局部频谱变化。

### 6.4 RGB 解码头

RGB 解码头负责恢复视觉质量，其目标包括：

- 消除由深度相关 PSF 引入的空间模糊；
- 保留纹理和颜色一致性；
- 尽量恢复边缘锐度；
- 减少伪影和振铃。

推荐结构仍以 U-Net 上采样路径为主，可配合：

- skip connection；
- 1x1 融合卷积；
- 残差块；
- 轻量注意力模块。

### 6.5 Depth 解码头

Depth 解码头负责从编码模糊模式中恢复几何结构。与 RGB 分支不同，它更关注：

- 模糊核的形状变化；
- 多尺度上下文；
- 边界与遮挡关系；
- 深度层之间的连续性。

它的输出可以有两种定义：

1. **度量深度**
   - 直接回归真实距离值。

2. **离散深度分类 + 回归修正**
   - 先预测深度层概率，再做细化回归。

在项目初期，直接回归单通道深度图实现更简单；后续可根据训练稳定性切换到分桶策略。

### 6.6 跨任务特征融合

单纯共享编码器往往不够。更有效的方式是在解码阶段引入跨任务交互，例如：

- 用深度分支特征引导 RGB 去模糊；
- 用 RGB 的边缘和纹理信息约束深度边界；
- 在 skip connection 处进行联合融合。

一个可行思路如下：

```python
rgb_feat = rgb_decoder_block(...)
depth_feat = depth_decoder_block(...)
fused = concat([rgb_feat, depth_feat, skip_feat])
fused = conv1x1(fused)
```

这样做的意义是让网络在边缘位置兼顾两件事：

- 图像边缘更清晰；
- 深度边界更准确。

### 6.7 与 `DeepOpticsHDR` 网络的关系

`DeepOpticsHDR` 的网络可作为 SMAID 的直接原型，尤其是：

- 编码器结构；
- skip connection 组织方式；
- 中等规模和大型规模网络模板；
- 反卷积与插值上采样模块。

SMAID 需要新增的主要内容是：

- 第二个解码头；
- 双任务特征交互；
- 深度输出层与相关归一化策略。

## 7. 损失函数设计

### 7.1 总体目标

SMAID 不是单目标优化，而是典型的多任务联合优化问题。总损失可写为：

$$
L_{total} = \lambda_{rgb}L_{rgb} + \lambda_{depth}L_{depth} + \lambda_{smooth}L_{smooth} + \lambda_{reg}R_{optics}
$$

其中四部分分别控制：

- 图像重建质量；
- 深度估计精度；
- 深度图空间平滑性；
- 相位板可制造性。

### 7.2 RGB 重建损失

对全聚焦 RGB 分支，推荐组合使用 `L1` 与 `SSIM`：

$$
L_{rgb} = \alpha (1 - SSIM(I, \hat{I})) + (1-\alpha)\|I-\hat{I}\|_1
$$

原因：

- `L1` 对边缘恢复更友好，通常比 `L2` 更少过平滑；
- `SSIM` 能补充结构相似性约束；
- 二者结合更适合去模糊和重建任务。

后续若追求感知质量，还可追加：

- perceptual loss；
- LPIPS；
- 频域一致性损失。

### 7.3 深度估计损失

对于深度分支，推荐优先采用 **SiLog** 或尺度不变对数损失：

$$
L_{depth} =
\sqrt{
\frac{1}{n}\sum_i e_i^2
- \frac{\lambda}{n^2}\left(\sum_i e_i\right)^2
}
,\quad
e_i = \log \hat{D}_i - \log D_i
$$

其优势在于：

- 弱化整体尺度偏移带来的影响；
- 更关注相对深度结构；
- 对深度估计中的尺度模糊更鲁棒。

如果任务要求严格度量深度，也可以尝试与以下项混合：

- `L1(\hat{D}, D)`
- `RMSE`
- `AbsRel`

### 7.4 边缘感知平滑损失

为了抑制深度图局部噪点，同时避免过度抹平边界，可引入边缘感知平滑项：

$$
L_{smooth} =
|\partial_x \hat{D}| e^{-|\partial_x I|}
+
|\partial_y \hat{D}| e^{-|\partial_y I|}
$$

意义是：

- 图像平坦区域鼓励深度平滑；
- 图像边缘位置允许深度不连续。

这对提升物体边界区域的深度质量非常重要。

### 7.5 光学正则化

为了保证相位板物理可制造，保留来自 `DeepOpticsHDR` 的高度图正则是合理的：

$$
R_{optics} = \|\Delta h\|_2^2
$$

其中 `\Delta` 为离散 Laplacian 算子。该项用于抑制高度图高频振荡。

### 7.6 可选辅助损失

在后续实验中可以逐步加入：

1. **深度边界一致性损失**
   - 强化 RGB 边缘与深度边界对齐。

2. **多尺度监督**
   - 对不同分辨率输出同时监督。

3. **频域损失**
   - 约束重建图像高频恢复。

4. **PSF 区分度正则**
   - 让不同深度层 PSF 在特征上保持可区分性。

## 8. 训练策略

### 8.1 训练目标

训练阶段需要联合优化两类参数：

- **光学参数**：相位掩模高度图；
- **网络参数**：RGB-D 多任务解码网络。

这要求整个前向过程从相位板到传感器再到网络输出都是可微的。

### 8.2 两阶段训练建议

参考 `DeepOpticsHDR` 的经验，建议采用两阶段策略：

#### 阶段一：端到端联合训练

- 使用仿真 `PSF_z`；
- 同时更新相位板和网络参数；
- 目标是学到既可编码深度又可恢复 RGB 的初始最优解。

#### 阶段二：固定或半固定光学参数微调

- 使用更真实的 PSF 或实测 PSF；
- 只微调网络，或对相位板使用很小学习率；
- 用于弥合仿真和真实制造之间的差距。

### 8.3 差异化学习率

光学参数与网络参数的尺度和敏感性不同，因此建议使用不同学习率：

```python
opt_net = Adam(network_lr)
opt_optics = Adam(optics_lr)
```

通常：

- `optics_lr` 可以比 `network_lr` 大或小，需要视具体梯度稳定性而定；
- 初期可尝试让相位板更新更慢，以避免训练早期陷入极端 PSF；
- 若发现网络主导优化、相位板几乎不动，再提高光学学习率。

### 8.4 数据增强

SMAID 推荐的数据增强包括：

- 随机裁剪；
- 水平翻转；
- 颜色抖动；
- 亮度 / 对比度扰动；
- 传感器噪声强度随机化；
- 深度范围扰动；
- 可选 JPEG / 压缩退化。

增强的目标是让网络不过度依赖某一种固定照明、纹理或噪声模式。

### 8.5 训练稳定性问题

SMAID 的训练相比普通图像任务更难，主要原因包括：

- 光学层引入的梯度传播更复杂；
- 深度相关 PSF 计算成本高；
- RGB 与深度任务之间存在潜在冲突；
- 多任务权重不当会导致一项任务压制另一项任务。

因此训练时建议重点监控：

- RGB 分支重建质量；
- 深度误差曲线；
- 不同深度层 PSF 的可分性；
- 相位板高度图是否出现异常尖峰；
- 梯度范数与显存使用情况。

## 10. 评估指标

### 10.1 全聚焦图像评估

对 RGB 输出建议关注以下指标：

- **PSNR**
  - 衡量像素级误差；
- **SSIM**
  - 衡量结构相似性；
- **LPIPS**
  - 衡量感知层面的视觉质量。

### 10.2 深度估计评估

对深度输出建议关注：

- **RMSE**
  - 反映整体误差幅度；
- **AbsRel**
  - 反映相对误差；
- **SqRel**
  - 强调大误差的影响；
- **\(\delta < 1.25\)**、**\(\delta < 1.25^2\)**、**\(\delta < 1.25^3\)**
  - 反映预测深度落入阈值范围的比例。

### 10.3 光学层分析指标

除最终任务指标外，SMAID 还建议分析光学层本身：

- 不同深度层 `PSF_z` 的互相关；
- PSF 的能量集中度与扩散范围；
- 各波长通道间的一致性；
- 相位板高度图的空间频率分布。

这些指标有助于理解模型到底学到了什么，而不仅仅是得到一个黑盒结果。

   - 多优化器管理方式

### 11.2 SMAID 需要新增的核心模块

1. **深度相关 PSF 生成模块**
   - 支持一次性生成多深度层 PSF。

2. **分层成像前向模块**

### 11.3 推荐开发顺序

为降低风险，建议采用以下顺序推进：

#### 第一步：搭建光学前向模块

- 从 `DeepOpticsHDR` 迁移 `optics.py` 基础能力；
- 实现 `compute_depth_dependent_psf()`；
- 验证不同深度层 PSF 是否有合理变化。

#### 第二步：实现分层成像模拟

- 输入全聚焦 RGB 与 GT depth；
- 实现多层卷积叠加；
- 输出模拟传感器图像；
- 确认前向模型数值稳定。

#### 第三步：先做固定光学、仅训练网络

- 暂时固定一个手工或随机初始化的相位板；
- 先验证网络能否从编码图像恢复 RGB 与 depth；
- 避免一开始就进行完整联合优化导致问题难定位。

#### 第四步：引入联合训练

- 在网络收敛基础上，开始更新相位板；
- 观察 PSF 变化是否提高深度与 RGB 性能；
- 调整损失权重和学习率。

#### 第五步：面向真实系统微调

- 加入制造误差模拟；
- 引入实测 PSF；
- 评估真实成像条件下性能退化与补偿空间。

## 12. 关键风险与现实挑战

### 12.1 计算成本

深度相关 PSF 和分层卷积会显著增加训练成本，特别是：

- 深度层数越多，前向代价越高；
- PSF 尺寸越大，卷积与显存开销越高；
- 联合优化会放大训练不稳定性。

因此在项目早期需要优先关注计算图规模控制。

### 12.2 任务冲突

适合深度编码的 PSF，不一定天然适合高质量 RGB 恢复；反之亦然。这意味着：

- 过强的深度编码可能导致 RGB 恢复变难；
- 过温和的 PSF 可能使深度信息不足。

这也是为什么 SMAID 必须采用多任务联合目标，而不能只优化某一项指标。

### 12.3 仿真到真实的差距

真实系统中会出现：

- 相位板加工误差；
- 安装偏心与倾斜；
- 传感器噪声偏离理想模型；
- 镜头系统色差与像差；
- 场景光谱分布与训练假设不一致。

这些因素都可能削弱仿真得到的最优相位板效果，因此必须在训练阶段预留鲁棒性设计。

### 12.4 深度边界难点

深度边界往往对应：

- 遮挡；
- 半透明区域；
- 高频纹理与低纹理混合；
- 一个像素内多深度混合。

这部分既是深度估计难点，也是全聚焦重建最容易出伪影的位置，需要在损失设计和网络融合上重点处理。

## 13. 当前项目结论

SMAID 可以被理解为在 `DeepOpticsHDR` 路线上的一次明确扩展：

- `DeepOpticsHDR` 关注的是**动态范围编码与恢复**；
- SMAID 关注的是**景深编码、全聚焦恢复与深度重建**。

从实现角度看，SMAID 最重要的不是重新发明一套光学-网络框架，而是把现有可微光学系统从“单一 PSF 的图像恢复问题”推进到“深度相关 PSF 的 RGB-D 联合恢复问题”。

因此，SMAID 的技术主线可以概括为：

1. 设计一个可制造、可微、具有深度区分能力的相位掩模；
2. 用深度相关前向模型把三维场景压缩成单张编码图像；
3. 用双头网络把该编码图像解码为高质量全聚焦 RGB 和深度图；
4. 通过端到端联合优化，使光学编码和神经解码彼此适配。

这份文档的意义在于把 SMAID 从“一个研究构想”整理为“一个可执行的工程蓝图”。后续代码开发将以这里定义的模块边界、前向模型和训练目标为基础逐步展开。
